Button

A clickable element that triggers an action, available in 3 types × 6 states with optional icon slots and loading state.

01

Anatomy

The Button is composed of 7 distinct parts. The container holds up to 5 visible elements — two icon slots, a text wrapper with label, plus two overlay elements (focus ring and spinner) that appear conditionally.

  • Container — <button> element. 32px height, 6px horizontal padding, 8px border-radius. Background and border vary by type and state.
  • Left Icon slot — 16×16px optional slot. Shown when showLeftIcon=true and state≠Loading. Defaults to Plus icon.
  • Text Wrapper — flex row with 6px horizontal padding centering the label.
  • Label — 14px/16px Medium text. Capitalized. Color: fg-primary for Primary/Secondary, fg-secondary for Tertiary. Hidden (opacity:0) during Loading.
  • Right Icon slot — 16×16px optional slot. Shown when showRightIcon=true and state≠Loading. Defaults to ChevronRight icon.
  • Focus Ring — absolute overlay, 2px border in #6388f7, border-radius 10px. Visible only in Focused state. Inset: -2px (Primary/Tertiary), -3px (Secondary).
  • Spinner — absolute centered 20×20px SVG. Visible only in Loading state. Replaces icon+label visually.

02

Variants & Style

Three visual variants establish a clear action hierarchy. Primary draws maximum attention, Secondary provides a contained alternative, and Tertiary is the lowest-emphasis option.

  • Primary — Filled style. Solid brand-green background (bg-brand-primary). Use for the main call-to-action on a screen. Maximum visual weight.
  • Secondary — Outlined style. Transparent background with a 1px border (border-secondary). Use as supporting action alongside a Primary button. Medium visual weight.
  • Tertiary — Ghost/Transparent style. No background, no border. Text only (+ icons). Use for low-priority or utility actions. Minimum visual weight.

04

States

The component responds to six interaction and system states. Every state applies to all three variants, yielding 18 distinct combinations.

  • Default — Rest state. Full opacity. No ring.
  • Hovered — Cursor over button. Darker background (Primary) or darker border (Secondary). Tertiary text becomes fg-primary.
  • Pressed — Active/click. Primary uses darkest bg (#008a52). Secondary border unchanged.
  • Focused — Keyboard focus. A 2px focus ring in #6388f7 appears outside the button. Styling same as Hovered otherwise.
  • Disabled — Not interactive. Opacity 0.4. Cursor: not-allowed. No hover effects.
  • Loading — Async action in progress. Spinner replaces label and icons. Button retains its type styling.

Themes

Light mode is fully implemented. Dark mode is not yet available — the token system does not define separate dark-mode values in tokens.css.

  • Light — Implemented. Default theme. White/neutral backgrounds, brand-green accents, near-black text.
  • Dark — Not implemented. No dark-mode CSS variables are defined in the current token set. All components will inherit light-mode values.
